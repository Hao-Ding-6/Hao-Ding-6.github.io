4. Median of Two Sorted Arrays

Binary search is very useful.

December 22, 2018 Hao Ding

5 minute read

  • Twitter

  • Google+

  • Facebook

  • Reddit

  • LinkedIn

  • StumbleUpon

  • Pinterest

  • Email

Description

There are two sorted arrays nums1 and nums2 of size m and n respectively.

Find the median of the two sorted arrays. The overall run time complexity should be O(log (m+n)).

You may assume nums1 and nums2 cannot be both empty.

Example 1:

nums1 = [1, 3]
nums2 = [2]

The median is 2.0

Example 2:

nums1 = [1, 2]
nums2 = [3, 4]

The median is (2 + 3)/2 = 2.5

Solution

To improve the time complecity to log-level, it is easy to think of binary search whose time complexity is O(log(n)).

Another important thing is that you should know what kind of role the median number plays in mathematics before solving this problem. In fact, median number is used to divide a set of number into two equal-length parts of which left part is always smaller than the right part.

Based on this definition, we can divide both nums1 and nums2 into two parts as the following shows. Let m = length of nums1, n = length of nums2, i is used to divide array nums1, j is used to divide array nums2.

            left_part                |              right_part
nums1[0], nums1[1], ..., nums1[i-1]  |  nums1[i], nums1[i+1], ..., nums1[m-1]
nums2[0], nums2[1], ..., nums2[j-1]  |  nums2[j], nums2[j+1], ..., nums2[n-1]

As we say above, the median number of these two array should devide the merged array into two equal-length parts, which means we can find appropriate positions for i and j that the left parts of nums1 and nums2 is as long as the right parts of nums1 and nums2, so we have:

# +1 is because the length of merged array might be odd, 
# in that case we should add 1 to the right part to be equal to the left part. 
i + j == m - i + n - j ( + 1)
# then do some transformation, we got:
# +1 is because it plays well in both odd and even case, it can guarantee the left part of merged array is longer than the right part in odd case wihch is very useful if you really understand the code below.
j = (m + n + 1) // 2 - i

Here we should notice i and j should not be negative and 0 <= i <= m, so to guarantee that j is not negative, we should let m <= n, wihch means if the length of nums1 is bigger than nums2, we should exchange the data.

If the merged array’s length is odd, the median number should be the maximum number of its left part (we guarantee that if the length of merged array is odd, the median number belongs to its left part); if even, the median number should be the average number of the maximum number of its left part and the minimum number of its right part. Therefore we have:

if odd:
    median = max_left
elif even:
    median = (max_left + min_right) / 2

Now, we can use binary search to find the position of i and get the corresponding position of j, for each position of i and j, check if the left part is always smaller than the right part, here we show the diagram again:

            left_part                |              right_part
nums1[0], nums1[1], ..., nums1[i-1]  |  nums1[i], nums1[i+1], ..., nums1[m-1]
nums2[0], nums2[1], ..., nums2[j-1]  |  nums2[j], nums2[j+1], ..., nums2[n-1]

The “left part is always smaller than the right part” means that nums1[i - 1] and nums2[j - 1] must be smaller than nums1[i] and nums2[j], because these two arrays are sorted, nums1[i - 1] and nums2[j - 1] are the maximum number of their left part, nums1[i] and nums2[j] are the minimum number of their right part.

If nums1[i - 1] is bigger than nums2[j], i should move to left (move to smaller part), if nums2[j - 1] is bigger than nums1[i], i should move to right (move to bigger part).

Finally we have the code:

class Solution:
    def findMedianSortedArrays(self, nums1, nums2):
        """
        :type nums1: List[int]
        :type nums2: List[int]
        :rtype: float
        """
        if len(nums1) > len(nums2):
            nums1, nums2 = nums2, nums1
        
        m = len(nums1); n = len(nums2)
        left = 0; right = m
        # i -> partition posision of array nums1
        # j -> partition position of array nums2 
        while left <= right:
            i = left + (right - left) // 2
            j = (m + n + 1) // 2 - i
            
            if i > 0 and nums1[i - 1]  > nums2[j]:
                right = i - 1
            elif i < m and nums1[i] < nums2[j - 1]:
                left = i + 1
            else:
                if i == 0:
                    max_left = nums2[j - 1]
                elif j == 0:
                    max_left = nums1[i - 1]
                else:
                    max_left = max(nums1[i - 1], nums2[j - 1])
                
                if (m + n) % 2 == 1:
                    return max_left
                
                if i == m:
                    min_right = nums2[j]
                elif j == n:
                    min_right = nums1[i]
                else:
                    min_right = min(nums1[i], nums2[j])
                
                return (max_left + min_right) / 2

If you are confused with the code, go back to this diagram and all problems will be solved:

            left_part                |              right_part
nums1[0], nums1[1], ..., nums1[i-1]  |  nums1[i], nums1[i+1], ..., nums1[m-1]
nums2[0], nums2[1], ..., nums2[j-1]  |  nums2[j], nums2[j+1], ..., nums2[n-1]
